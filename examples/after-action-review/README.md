# After Action Review Examples

Use this folder for sanitized examples of after action review memos.

Good examples should include:

- Event reviewed
- Expected outcome
- Actual outcome
- What went well
- What went wrong
- Why it happened
- Pattern matches
- Lessons
- System changes
- Follow-up actions with owner, due date, and validation method
- Leader notes

Initial examples to add later:

- Second product launch with unexpected support load
- Customer-facing migration that slipped and created confusion
