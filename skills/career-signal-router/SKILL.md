# Career Signal Router

## Purpose

Route career activity into no-regrets actions, option-preserving bets, portfolio assets, and decision gates.

This skill helps a career transition stay concrete without overcommitting too early or drifting into vague exploration.

## Context Check

Before using this skill:

- Identify the user's current career objective and time horizon.
- Identify the main candidate paths, roles, domains, or projects.
- Clarify constraints such as location, compensation, timeline, energy, and risk tolerance.
- Separate exploration from commitment.
- Check whether the output should be a plan, scorecard, weekly actions, or decision memo.

## When To Use

Use this skill when:

- A person is exploring multiple career paths or role shapes.
- A portfolio project should support hiring, founder, consulting, or learning outcomes.
- The next actions should create useful signal across several possible futures.
- A decision date or commitment gate is approaching.

Do not use it for resume editing, generic job-search advice, or company research.

## Inputs

Useful inputs include:

- Current career criteria
- Candidate roles, sectors, or companies
- Portfolio projects or learning tracks
- Constraints and non-negotiables
- Timeline and decision dates
- Existing conversations, applications, or market signals

## Workflow

1. **Map the live options**
   - Name each active path and what would make it viable.

2. **Identify no-regrets actions**
   - Choose actions that help across multiple paths, such as building a portfolio artifact, having a domain conversation, or tightening positioning.

3. **Route path-specific bets**
   - Mark actions that only pay off if one path remains attractive.
   - Keep these smaller until stronger signal arrives.

4. **Define portfolio assets**
   - Identify concrete artifacts that prove capability and can be shown publicly or privately.

5. **Set signal questions**
   - For each path, define what evidence would increase, reduce, or end conviction.

6. **Create decision gates**
   - Define dates, criteria, and choices.
   - Avoid permanent ambiguity by naming when exploration becomes commitment, deferral, or shutdown.

## Output Structure

Use this structure:

```md
# Career Signal Router

## Live Options
| Path | Why It Matters | What Must Be True |
| --- | --- | --- |

## No-Regrets Actions
- Action, why it helps multiple paths, next step.

## Path-Specific Bets
- Action, path served, cost, signal expected.

## Portfolio Assets
- Artifact, audience, proof it provides, deadline.

## Decision Gates
| Date | Decision | Criteria |
| --- | --- | --- |
```

## Quality Bar

A good routing plan:

- Creates forward motion without premature commitment.
- Distinguishes broad signal from narrow bets.
- Produces concrete actions and artifacts.
- Defines the evidence needed to make or close decisions.
- Keeps constraints visible instead of treating them as footnotes.

## Common Failure Modes

- **Generic career advice:** the output must route real options and actions.
- **All exploration, no gates:** open-ended exploration becomes drift.
- **Overbuilding portfolio work:** a portfolio asset should prove a capability, not become an endless product.
- **Ignoring constraints:** compensation, location, timing, and energy change the real option set.
