# Learning Loop Designer

## Purpose

Design a repeatable learning loop that teaches through short lessons, active practice, immediate feedback, spaced review, and real-world application.

This skill is for building learning systems, not one-off explanations.

## Context Check

Before using this skill:

- Identify the learner, domain, and current level.
- Clarify the job the learning loop must support.
- Separate concepts to understand from behaviors to practice.
- Identify the cadence and channel constraints.
- Check whether the domain requires current-source verification, expert review, or safety boundaries.

## When To Use

Use this skill when:

- A topic needs recurring practice rather than a single explanation.
- The learner needs durable understanding, not passive content consumption.
- A product, course, study routine, or coaching flow needs lesson structure.
- The output should include questions, feedback, review, and application.

Do not use it when a direct answer, memo, or reading list is enough.

## Inputs

Useful inputs include:

- Learner profile and baseline knowledge
- Learning goal and practical use case
- Domain constraints and safety issues
- Available channels such as web, email, Slack, chat, or document
- Topic list, glossary, or source material
- Desired cadence and session length

## Workflow

1. **Define the capability**
   - State what the learner should be able to do, explain, judge, or apply.

2. **Break the topic into loops**
   - Each loop should fit one small concept plus one active practice round.
   - Avoid cramming a full lesson plan into one session.

3. **Choose the practice modes**
   - Use multiple choice for recall.
   - Use scenario judgment for tradeoffs.
   - Use ordering or matching for process and relationships.
   - Use short written application for real-world transfer.

4. **Design feedback**
   - Give immediate feedback after each answer.
   - Explain why the best answer is best and why attractive wrong answers fail.

5. **Add spaced review**
   - Revisit weak or important concepts at increasing intervals.
   - Keep review lightweight and targeted.

6. **Connect to application**
   - End each loop with one action, observation, or reflection tied to the learner's real context.

## Output Structure

Use this structure:

```md
# Learning Loop Plan

## Learner And Goal
- Audience, current level, capability target.

## Loop Map
| Loop | Concept | Practice Mode | Application |
| --- | --- | --- | --- |

## Sample Lesson
- Hook
- Micro-insight
- Practice questions
- Feedback
- Apply-it prompt

## Review Plan
- What repeats, when, and why.

## Quality Checks
- How to tell whether the learner is improving.
```

## Quality Bar

A good learning loop:

- Teaches one usable concept at a time.
- Makes the learner do something with the concept.
- Gives feedback that changes understanding.
- Revisits important material before it decays.
- Connects learning to real behavior or judgment.

## Common Failure Modes

- **Content dump:** too much explanation without active recall or application.
- **Trivia questions:** questions test vocabulary but not judgment.
- **No feedback:** answer keys without explanation do not teach.
- **No cadence:** learning loops need recurrence to build retention.
- **Unsafe confidence:** regulated or safety-critical domains need source checks and expert boundaries.
